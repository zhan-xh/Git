github 使用版本控制

基本命令提交练习

git init 创建.git目录用来存放复制文件
git add +路径 git add .表示提交当前目录
.gitignore 不需要提交的文件
git commit提交
git commit m "版本1"
git log查看修改的版本
git reset --hard 版本切换
git reflog 看所有历史

增加了分支的使用 git branch X 基于当前commit创建一个新的时间线 git checkout x 用于切换分支 git branch -d x 当分支无用时，删除分支 git branch 可以查看当前所有分支

对于git hub的使用：1.创建本地仓库，并提交代码 2.github创建远程仓库 3.复制远程仓库给出的地址，在vscode终端运行 提交完成 github主要是解决程序员的版本控制，以及多分支同时开发的问题 平时在编写代码的时候，每次修改后直接提交本地仓库，写明发现了什么bug，修改了什么bug，最后项目完成时，就提交github，修改的记录都会呈现
